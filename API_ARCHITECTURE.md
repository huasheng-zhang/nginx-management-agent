## 组件说明

### 1. 客户端层 (Client Layer)
- **Web UI**: 基于浏览器的管理界面
- **Mobile App**: 移动设备管理应用
- **CLI Tool**: 命令行管理工具
- **Scripts**: 自动化脚本

### 2. API网关层 (API Gateway)
- **Flask Application**: 核心Web服务
- **认证中间件**: Token验证
- **CORS配置**: 跨域访问控制
- **日志系统**: 请求/响应日志记录

### 3. 业务逻辑层 (Business Logic)
- **Nginx服务层**: 进程控制（启动、停止、重载）
- **配置管理层**: 配置文件的CRUD操作和备份管理
- **日志管理层**: 日志文件查看和管理

### 4. 数据/系统层 (Data/System Layer)
- **Nginx二进制**: /usr/sbin/nginx
- **配置文件**: /etc/nginx/nginx.conf
- **日志文件**: /var/log/nginx/
- **备份存储**: /etc/nginx/backup/
- **系统监控**: CPU、内存、磁盘、进程

## 数据流图

### 1. 配置更新流程

```
客户端
   │
   │ PUT /nginx/config
   │ {"content": "new config"}
   ▼
API服务
   │
   ├─► 认证检查
   │
   ├─► 备份当前配置
   │   └─► /etc/nginx/backup/
   │
   ├─► 测试新配置
   │   └─► nginx -t
   │
   ├─► 写入配置文件
   │   └─► /etc/nginx/nginx.conf
   │
   └─► 返回结果
       └─► {"success": true}
```

### 2. 日志查看流程

```
客户端
   │
   │ GET /nginx/logs/error.log?lines=100
   ▼
API服务
   │
   ├─► 认证检查
   │
   ├─► 验证文件路径
   │   └─► /var/log/nginx/
   │
   ├─► 读取文件尾部
   │   └─► tail -n 100
   │
   └─► 返回日志内容
       └─► {"content": "..."}
```

### 3. 进程控制流程

```
客户端
   │
   │ POST /nginx/reload
   ▼
API服务
   │
   ├─► 认证检查
   │
   ├─► 检查Nginx状态
   │   └─► psutil检查进程
   │
   ├─► 执行命令
   │   └─► nginx -s reload
   │
   ├─► 验证结果
   │
   └─► 返回状态
       └─► {"success": true}
```

## 配置管理流程

```
配置更新请求
   │
   ├─► 1. 接收新配置
   │   └─► 请求体: {"content": "..."}
   │
   ├─► 2. 备份当前配置
   │   ├─► 创建备份文件
   │   ├─► 命名: nginx.conf.YYYYMMDD_HHMMSS
   │   └─► 存储: /etc/nginx/backup/
   │
   ├─► 3. 测试新配置
   │   ├─► 写入临时文件
   │   ├─► 执行: nginx -t
   │   ├─► 成功 → 继续
   │   └─► 失败 → 返回错误
   │
   ├─► 4. 应用配置
   │   └─► 写入: /etc/nginx/nginx.conf
   │
   └─► 5. 返回结果
       └─► {"success": true, "backup_path": "..."}
```

## 版本对比：v1 vs v2

### v1架构（单体）
```
app.py
   │
   └─► 所有路由和逻辑在一个文件
       ├─► 简单直接
       ├─► 适合小型部署
       └─► 配置硬编码
```

### v2架构（模块化）
```
app_v2.py
   │
   ├─► 应用工厂
   ├─► 配置管理
   └─► 蓝图注册
       │
main.py（蓝图）
   │
   └─► 路由定义
   └─► 业务逻辑
       │
config.py
   │
   └─► 环境配置
   └─► 多环境支持
```

**v2优势**：
- ✅ 模块化设计，易于维护
- ✅ 配置灵活，支持多环境
- ✅ 安全性增强（认证、CORS）
- ✅ 可扩展性好
- ✅ 日志系统完善
